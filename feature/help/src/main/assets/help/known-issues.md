# 已知问题

本文基于当前源码整理，尽量只保留现在仍然成立、而且用户确实会遇到的限制与风险点。

## 1. 大型项目性能压力明显

### 现象

- 大型 C/C++ 项目在手机或平板上构建会明显变慢
- clangd 分析大型工程时内存占用较高
- 多个大文件同时打开时，编辑体验可能下降

### 建议

1. 优先使用增量构建
2. 对大型工程考虑使用 **远程 LSP**
3. 不要一次打开过多大文件

## 2. 首次启动与开发环境部署耗时较长

### 现象

- 首次启动或版本升级后，开发环境准备时间可能较长
- 在开发环境尚未就绪前，终端 / 调试 / 构建能力可能不可用

### 建议

1. 首次启动时尽量保持网络稳定
2. 确保存储空间充足
3. 若安装过程异常，进入设置重新部署开发环境

## 3. 外部工作空间与 SAF 在小文件密集场景下会更慢

### 现象

- 外部存储目录通过 SAF 访问时，频繁的小文件操作可能不如内部路径流畅
- 大型仓库或层级很深的工程，目录浏览和扫描可能更慢

### 建议

1. 工作空间尽量保持结构清晰
2. 对频繁构建的项目，尽量避免无关的大量临时文件
3. 出现明显卡顿时，先排查是否是外部存储 I/O 瓶颈

## 4. PRoot / 终端 / 调试链路受设备兼容性影响

### 现象

- 某些 ROM、SELinux 策略或设备环境会影响 PRoot 行为
- 独立终端有时可能启动失败
- LLDB 调试链路在少数设备上可能不稳定

### 建议

1. 遇到启动失败时先重新部署开发环境
2. 先验证普通运行流程，再验证调试流程
3. 保留日志，便于后续定位问题

## 5. 快捷键主要面向硬件键盘

### 现象

- 软键盘往往不能稳定触发 `Ctrl`、`Shift`、`F2` 这类组合键
- 编辑器快捷键在硬件键盘上的体验明显更好

### 建议

1. 把快捷键功能理解为“硬件键盘增强”
2. 若依赖快捷键工作流，优先连接蓝牙或 USB 键盘

## 6. 许可证列表以应用内打包清单为主

### 现象

- 开源许可证页主要展示当前应用依赖与部分打包分发组件
- 个别源码级、原生级子模块可能不会以单独 Maven 条目的形式出现

### 建议

1. 先查看应用内 **开源组件与许可证** 页面
2. 如需核对更底层的源码组件，请同时参考仓库内随附的 `LICENSE` / `NOTICE` 文件

## 7. 系统兼容性边界

### 当前最低要求

- **Android 9.0 / API 28**

### 建议运行环境

- 推荐 **3GB 以上内存**
- 推荐保留足够的可用存储空间，用于工作空间、工具链与构建产物

## 8. 开源版能力与常见疑问

### 在线能力

开源版仍保留反馈、插件浏览、插件安装、包下载、收藏、下载历史和 GitHub Release 版本更新提示。

### 私有身份和商业功能

私有身份、商业功能、私有后端和管理端不随公开仓库分发。

### AI 使用方式

AI 通过 Custom BYOK 渠道使用。请在 AI 设置中配置自己的服务商地址、模型和 API Key。

### 插件市场

开源版市场读取 GitHub Registry。客户端负责浏览、安装和更新插件，不包含私有账号互动和审核流程。

## 9. NDK 共享库模板运行结果看起来像“旧模板”

### 现象

- 新建 NDK 共享库项目后，粘贴新代码
- 编辑器里已经显示新内容
- 点击运行后，控制台还是打印模板测试程序的输出

### 说明

这通常不是“编译器没有重新编译源码”，而是：

1. NDK 共享库模板本来就同时带有共享库目标和测试可执行文件
2. 默认运行配置常常优先指向测试目标
3. 你看到的输出来自测试程序，不一定来自共享库本体

### 建议

1. 先确认文件是否已经保存到磁盘
2. 检查当前运行配置选择的是哪个目标
3. 如果目标是共享库，请切换到 SDL 运行模式或显式选择共享库目标
4. 重新打开项目后高亮恢复，通常说明语言服务重新加载了文件，不代表运行目标自动切换了

### 常见误解

- “我改了源码，运行还是旧的” 不一定是缓存问题
- “高亮能显示新代码” 不等于“当前运行目标就是新代码”
- “重新构建报错了” 也可能只是原先默认跑的目标和你现在想验证的目标不一样

## 10. 构建进程超时或取消后的资源清理

### 说明

构建使用的 CMake、Make、clang 等工具会作为短生命周期进程执行。正常情况下，构建结束、超时或取消后，TinaIDE 会关闭输出流、终止仍在运行的构建进程，并删除本次命令使用的临时目录。

### 不会自动清理的内容

- 项目 `build` 目录
- `compile_commands.json`
- 增量构建缓存
- 已成功生成并注册的 artifact

这些文件属于后续构建和源码变更检测需要复用的持久产物。如果怀疑持久缓存异常，请优先使用“清理构建目录 / 清理并重新配置”，不要手动删除不明确的内部目录。
