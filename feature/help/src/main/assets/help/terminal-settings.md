# 终端设置说明

本文基于当前 **设置 → 终端** 页面源码整理，重点说明主题、字体、后端、Shell、Linux 工具和交互项这些真实存在的设置。

## 当前终端设置页分成哪几块

现在的页面主要分成五组：

- **终端主题**
- **字体**
- **Shell 配置**
- **Linux 工具**
- **交互行为**

其中：

- **Linux 工具** 这一组只有在 Linux 开发环境启用时才会显示

所以如果你在某些设备或某些阶段没看到这一组，不一定是界面缺了，很可能只是当前环境还没启用。

## 终端主题

这一组现在只有一项：

- **主题**

当前主题列表来自终端主题提供器，不是写死在页面里的固定字符串表。

也就是说，终端里最终能看到哪些主题，要以当前主题提供器给出的实际列表为准。

## 字体

这一组当前有两项：

- 字体
- 字体大小

### 字体

当前字体入口不是简单的二选一，而是动作面板。

它会根据当前状态提供这些选择：

- 使用内置字体
- 使用系统等宽字体
- 使用自定义字体
- 选择 / 更换自定义字体文件

### 内置字体

内置字体不一定总是存在。

如果当前设备上没有对应内置字体，页面会直接显示“未安装”的状态，而不是假装它可用。

### 自定义字体

当前通过系统文件选择器导入字体文件。

导入后会调用终端偏好设置做校验：

- 有效字体才会真正保存
- 无效字体会直接失败

所以如果你选了文件却没成功，优先怀疑字体文件本身，而不是设置页没有保存。

### 字体大小

当前是滑块，范围是：

- `8sp`
- 到
- `32sp`

如果你觉得终端字符太挤、太大，先调这个值。

## Shell 配置

这一组当前有两项：

- 终端后端
- Shell 类型

### 终端后端

当前可选值有：

- **AUTO**
- **PROOT**
- **HOST**

怎么理解：

- **AUTO**：如果已经装了 PRoot，就优先走 PRoot；否则走 HOST
- **PROOT**：强制走 Linux 容器环境
- **HOST**：强制走 Android 宿主环境

### 一个很容易忽略的细节

如果 Linux 开发环境没有启用，而你之前又把后端存成了 `PROOT`，当前设置页会把它强制纠正回 `HOST`。

所以在“没有 Linux 环境”的前提下，别把 `PROOT` 当成长期可用值。

### 切换后端什么时候生效

当前文案明确说明：

- **需要重启终端生效**

### Shell 类型

当前可选值有：

- **AUTO**
- **sh**
- **bash**
- **zsh**

其中自动检测的优先级是：

- `zsh > bash > sh`

### 选 zsh 时会发生什么

当前逻辑不是简单地直接保存：

- 如果 zsh 已可用，就正常切换
- 如果当前在 PRoot 后端但 zsh 不可用，会引导安装
- 如果当前在 HOST 后端，zsh 不可用时会直接提示不支持

所以“切到 zsh 没成功”时，先看自己到底是在 PRoot 还是 HOST。

### 切换 Shell 什么时候生效

和后端一样：

- **需要重启终端生效**

## Linux 工具

这一组是终端设置里最偏“环境诊断”的区域。

当前会显示：

- Linux 工具健康状态
- 当前系统名称
- 当前包管理器
- Guest Dev Packages
- Zsh
- Locale 支持

### Linux 工具健康状态

这个值不是单一布尔，而是综合结果。

当前会根据这些状态判断：

- Guest Dev Packages
- Zsh
- Locale 支持

但有一个例外：

- 如果当前 Locale 是 `C.UTF-8`，则 Locale 支持不会被算作必需项

### 当前包管理器

当前会把包管理器显示成这些标签之一：

- `APK`
- `APT`
- `PACMAN`
- `DNF`
- `UNKNOWN`

### Guest Dev Packages

这一项代表一组终端开发命令是否齐全。

当前会检查的命令组包括：

- `cc / gcc / clang`
- `make`
- `git`
- `curl`
- `cmake`
- `pkg-config / pkgconf`

如果缺失，页面不只是显示“未就绪”，还会在动作面板里给出更细的缺失诊断。

### Guest Dev Packages 当前可执行的动作

根据状态不同，页面可能给你：

- 安装
- 重新安装

### Zsh

这一项除了显示状态，还支持：

- 重新安装
- 跳去修改 Shell

所以它并不只是一个“只读状态位”。

### Locale 支持

这一项用于保证当前终端语言环境真的有对应语言包。

当 Locale 不是 `C.UTF-8` 时，如果系统缺对应支持，页面会给你安装或重建入口。

## 交互行为

这一组当前有这些项目：

- Locale
- 光标闪烁
- 闪烁速率

### Locale

当前可选值是：

- `C.UTF-8`
- `zh_CN.UTF-8`
- `zh_TW.UTF-8`
- `en_US.UTF-8`
- `ja_JP.UTF-8`

如果你选的不是 `C.UTF-8`，当前逻辑会先检查：

- 是否需要先安装对应 Locale 支持

也就是说，改值不一定会立刻直接保存，有时会先进入安装流程。

### 光标闪烁

这是一个开关。

只有当它开启时，页面才会继续显示：

- **闪烁速率**

### 闪烁速率

当前是滑块，范围是：

- `100 ms`
- 到
- `2000 ms`

如果你不喜欢光标太跳，可以直接调低频率，或者直接关闭闪烁。

## 什么时候建议怎么配

### 想稳一点

优先考虑：

- 后端：`AUTO`
- Shell：`bash` 或 `AUTO`
- Locale：`C.UTF-8`

### 想要完整 Linux 命令链

优先检查：

- 后端：`PROOT` 或 `AUTO`
- Guest Dev Packages：已安装
- Zsh：按需要安装

### 想解决中文显示问题

优先检查：

1. Locale 当前值
2. Locale 支持是否已安装
3. 字体是否适合显示对应字符

## 常见问题

### 为什么设置里没有 PRoot 选项，或者总被切回 HOST

因为当前没有启用 Linux 开发环境时，`PROOT` 不是稳定可用状态，设置页也会把不合法的持久值纠正回 `HOST`。

### 为什么改了 Shell 或后端，当前终端会话没变

因为这两类设置都需要：

- **重启终端生效**

### 为什么导入自定义字体失败

最常见原因就是字体文件无效，或者不符合终端字体校验要求。
