# 编辑器设置说明

本文基于当前编辑器设置页源码整理，重点说明字体、主题、显示、缩进和性能相关选项。

## 编辑器设置页现在分成哪些块

当前页面分成四组：

- **字体**
- **显示**
- **编辑行为**
- **性能**

如果你只是觉得“编辑器看起来不顺眼”或“输入手感不对”，先从这四组里定位。

## 字体

当前字体设置主要包括：

- 字体大小
- 自定义字体

### 字体大小

当前滑块范围对应：

- 最小 `8sp`
- 最大 `48sp`

如果你在平板、大屏设备或高分辨率设备上编辑，这一项影响非常直接。

### 自定义字体

当前页面支持：

- 恢复默认字体
- 从文件选择自定义字体

设置时会先复制字体文件，再做有效性校验；如果字体文件本身无效，当前实现会直接拒绝，不会强行保存一个坏路径。

如果你切换字体失败，先怀疑字体文件本身，而不是设置没写进去。

## 显示

当前显示类设置包括：

- 编辑器主题
- 显示行号
- 自动换行
- 彩虹括号
- 彩虹括号最大行数
- 代码折叠
- 空白字符显示模式

### 编辑器主题

当前内置主题值包括：

- GRAY
- DARK
- LIGHT
- AUTO

除此之外，源码里也允许出现：

- **插件主题**

也就是说，如果你装了主题型插件，设置页里显示的主题来源不一定只有内置项。

### 彩虹括号最大行数

这是一个单独的输入项，不只是简单开关。

当前有效范围是：

- `0` 到 `200000`

其中：

- `0` 表示 **不限**

如果你担心大文件性能，可以适当限制它，而不是直接关掉彩虹括号。

### 空白字符显示模式

当前支持：

- **不显示**
- **仅边界**
- **全部显示**

如果你主要关心缩进边界和尾部空格，通常不用直接开到“全部显示”。

## 编辑行为

这一组包含：

- 自动缩进
- 用空格代替 Tab
- Tab 宽度

### Tab 宽度怎么改

当前实现不是弹窗输入，而是点击后在几个固定值之间轮换：

- `2`
- `4`
- `8`

所以你每点一次，值会直接跳到下一个预设，而不是让你手输任意数字。

### 用空格代替 Tab

这个开关会影响你按下 Tab 后写入的内容形式。

如果你和团队已经约定了缩进风格，最好和项目实际格式保持一致，不要只按个人偏好乱改。

## 性能

当前这一组主要是：

- **编辑器硬件加速**

如果你在某些设备上遇到：

- 渲染卡顿
- 滚动体验不稳
- 个别设备兼容性问题

这个开关值得单独测试。

## 哪些设置最常影响“看起来不对”

优先排查这些：

- 主题
- 字体大小
- 自定义字体
- 空白字符显示
- 行号
- 自动换行

## 哪些设置最常影响“编辑手感不对”

优先排查这些：

- 自动缩进
- 用空格代替 Tab
- Tab 宽度
- 硬件加速

## 常见问题

### 为什么我选了字体文件还是没生效

先检查：

1. 字体文件本身是否有效
2. 文件是否真的被导入成功
3. 当前是不是已经恢复回默认字体

### 为什么 Tab 宽度不是输入框

因为当前实现就是固定在几个常用预设值之间轮换：

- 2
- 4
- 8

### 为什么我已经开启彩虹括号，但大文件里表现还是有限

先检查：

- 彩虹括号最大行数是不是被限制住了

如果该值太小，超过范围后就不会继续完整渲染。
