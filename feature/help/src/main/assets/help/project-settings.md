# 项目设置说明

本文基于当前项目设置页源码整理，重点说明哪些选项是全局偏好，哪些选项必须在“有项目上下文”时才会显示。

## 先理解项目设置页的显示逻辑

当前项目设置页并不是所有入口都一直显示。

可以先把它分成两类：

- **全局偏好**
  - 自动保存
  - 自动备份
  - 新建项目默认源码位置
- **项目专属配置**
  - 当前项目
  - APK 导出类型
  - 原生依赖路径
  - 原生构建标志

如果你是从主页齿轮直接进入，而且当前没有打开项目，那么后一类内容本来就不会出现。

## 全局偏好部分

这一部分只有在不是“指定项目覆盖模式”时才会显示。

### 自动保存

当前支持的间隔选项包括：

- 关闭
- 30 秒
- 60 秒
- 5 分钟

如果你想降低频繁写盘，可以把它调大；如果你经常改动很多文件又怕忘记保存，可以适当调小。

### 自动备份

这是一个独立开关。

适合场景：

- 你经常做实验性修改
- 不希望因为误操作丢掉编辑结果

如果你已经有非常明确的 Git 工作流，也可以根据习惯决定是否长期开启。

### 新建项目默认源码位置

当前支持两种来源位置：

- **Public**
- **Private**

这影响的是以后新建项目时默认把源码放到哪里，而不是立即移动当前已存在的项目。

## 项目专属配置什么时候出现

源码里只有在当前存在项目上下文时，下面这些分类才会显示：

- 项目总览
- 原生依赖路径
- 原生构建标志

也就是说，如果你看不到这些分类，先别怀疑文档过时，先确认当前是不是：

- 已打开项目
- 或者从项目列表的“项目设置”入口进入

## 项目总览

当前项目分类里主要有这些能力：

- 查看当前项目名称
- 设置 APK 导出类型
- 重新识别 APK 导出类型

### APK 导出类型

当前支持的类型包括：

- Native Activity
- SDL3
- Terminal
- Disabled

如果项目不是明确的 APK 导出模板工程，页面也可能显示：

- **未检测到**

### 重新识别什么时候会出现

源码里这一项只会在**不是目标项目覆盖模式**时显示。

也就是说：

- 正常会话项目：可以看到重新识别
- 从项目列表对某个项目直接进设置：这项可能不显示

这是当前设计限制，不是少渲染了。

## 原生依赖路径

当前页面支持管理三类路径：

- Include 目录
- Library 目录
- Runtime 目录

页面本身先显示的是“数量摘要”，点击后进入编辑对话框，再按行维护具体路径。

### 什么时候该改它

适合场景：

- 头文件搜索路径不完整
- 链接库目录没进构建参数
- 运行时依赖库目录需要显式配置

如果你已经在 CMakeLists.txt 里把路径配全了，这里不一定非要再写一遍；但当项目元数据和构建配置需要补充时，这里很有价值。

## 原生构建标志

当前支持这几类：

- CFLAGS
- CXXFLAGS
- LDFLAGS
- LDLIBS
- CMake Args

### 它们的编辑方式不完全一样

- `CFLAGS / CXXFLAGS / LDFLAGS / LDLIBS`
  - 按文本输入
  - 最终会做归一化，按空格合并
- `CMake Args`
  - 按多行列表维护
  - 更适合逐项管理参数

如果你发现自己输入很多换行但保存后变成一行，这是当前实现的正常行为，不是丢数据。

## 路径和标志值在列表里怎么看

当前列表页不会直接把所有内容都展开，而是会做摘要展示：

- 路径类：显示条目数量
- 标志类：显示截断后的文本摘要

这是为了让设置页本身保持可读，不是数据没保存。

## 哪些场景最值得看项目设置

建议优先打开这一页的场景：

- 构建能跑，但头文件仍然报找不到
- 链接参数需要补充
- 想给某个项目明确指定 APK 导出模板
- 想统一新项目的默认源码位置

## 常见问题

### 为什么我看不到“项目总览 / 原生依赖路径 / 原生构建标志”

先检查：

1. 当前是否真的打开了项目
2. 当前是不是从主页设置直接进入

没有项目上下文时，这三块本来就不会渲染。

### 为什么我改了新建项目默认源码位置，当前项目没变化

因为这个设置影响的是：

- **以后新建的项目**

而不是当前已经存在的项目目录。

### 为什么列表里只显示“3 项”或一小段文本

因为当前设置页对路径和构建标志做的是摘要显示。

真实内容需要点进去编辑器对话框查看。
