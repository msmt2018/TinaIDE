# 快捷键设置说明

本文基于当前快捷键设置页源码整理，重点说明当前能自定义哪些动作、如何录入快捷键，以及当前系统的限制。

## 快捷键设置页现在能改什么

当前实现不是桌面 IDE 那种上百项快捷键系统，而是聚焦在少量高频动作上。

现在可自定义的动作分成四组：

- **文件操作**
  - 保存当前文件
  - 保存全部文件
- **标签页**
  - 关闭当前标签
  - 关闭全部标签
  - 下一个标签
  - 上一个标签
- **编辑**
  - 撤销
  - 重做
- **书签**
  - 切换当前书签
  - 下一个书签
  - 上一个书签

另外还有：

- **恢复默认快捷键**

## 默认快捷键是什么

当前默认映射包括：

- `Ctrl + S`：保存当前文件
- `Ctrl + Shift + S`：保存全部文件
- `Ctrl + W`：关闭当前标签
- `Ctrl + Shift + W`：关闭全部标签
- `Ctrl + Tab`：下一个标签
- `Ctrl + Shift + Tab`：上一个标签
- `Ctrl + Z`：撤销
- `Ctrl + Shift + Z`：重做
- `Ctrl + F2`：切换当前书签
- `F2`：下一个书签
- `Shift + F2`：上一个书签

如果你只是偶尔使用快捷键，其实只记住前几组高频动作就够了。

## 怎么修改某个快捷键

当前设置页的流程是：

1. 点某个动作
2. 进入“捕获按键”对话框
3. 按下新的组合键
4. 确认保存

当前对话框还支持：

- 单独恢复该动作的默认快捷键

所以你不需要先全局重置，才能修正某一个动作。

## 当前支持哪些修饰键

源码里当前捕获逻辑会记录：

- `Ctrl`
- `Shift`
- `Alt`

如果只是单独按修饰键本身，例如只按 `Ctrl` 或只按 `Shift`，当前实现不会把它当成一个有效快捷键。

这能避免把“只有修饰键”的无意义组合保存进去。

## 冲突检测怎么工作

当你给某个动作设置新快捷键时，页面会检查它是否已经被其他动作占用。

如果冲突，会直接提示：

- 当前这个组合键已经被哪个动作使用

也就是说，当前快捷键系统不会默默覆盖旧映射。

## 为什么强调硬件键盘

当前设置页本身就写明了一个关键前提：

- **需要硬件键盘**

因为快捷键捕获依赖真实按键事件。很多软键盘并不会稳定发出：

- `Ctrl`
- `Shift`
- `Alt`
- `F2`

这些组合。

如果你主要用的是触屏软键盘，就不要把这套系统当成主要输入方式。

## 什么场景最值得用快捷键设置

适合场景：

- 你经常接蓝牙键盘或键盘壳
- 习惯桌面 IDE 式的操作节奏
- 经常在多个标签和书签之间切换

不太适合场景：

- 完全不用硬件键盘
- 只偶尔保存一次文件

## 恢复默认快捷键

当前支持两种恢复方式：

- 在单个动作对话框里恢复默认
- 在设置页底部恢复全部默认快捷键

如果你只是改坏了一个快捷键，优先用单项恢复，不要一上来全部重置。

## 当前系统的边界

建议直接记住这几点：

- 当前只覆盖少数高频动作
- 当前主要面向硬件键盘
- 当前依赖 Android 按键事件
- 当前支持冲突检测，但不支持桌面 IDE 那种复杂上下文规则

所以如果你想找“几百项快捷键映射表”，当前实现并不是这个方向。

## 常见问题

### 为什么按了组合键却录不进去

先检查：

1. 当前是不是在“捕获按键”状态
2. 是否真的接了硬件键盘
3. 你按的是不是只有修饰键，没有实际主键

### 为什么软键盘下快捷键不稳定

因为很多软键盘并不会完整上报桌面式组合键事件，这属于输入方式限制，不是设置页本身的 bug。

### 为什么我想改的动作不在列表里

因为当前实现只开放了少数高频动作，并不是完整桌面 IDE 快捷键系统。
