## 开发环境设置

### 必需工具

1. **Android Studio** (最新稳定版)
2. **JDK 17+**
3. **Docker Desktop**
4. **PowerShell 7+**
5. **Git**

### 项目设置

```bash
# 克隆项目
git clone https://github.com/wuxianggujun/TinaIDE.git
cd TinaIDE

# 构建工具链
pwsh ./docker/llvm-build/build-local.ps1 -Abi arm64-v8a -ApiLevel 24

# 同步到项目
pwsh ./tools/sync-llvm-build.ps1 -Abi arm64-v8a -ApiLevel 24

# 在 Android Studio 中打开项目
```

#### 函数规范

```kotlin
// ✅ Good: 简洁清晰
fun loadLibrary(name: String) {
    System.loadLibrary(name)
}

// ✅ Good: 使用命名参数
fun compile(
    source: String,
    target: String = "aarch64-linux-android24",
    optimize: Boolean = true
)

// ✅ Good: 使用表达式函数
fun isValidProject(path: File): Boolean = 
    path.exists() && path.isDirectory

// ❌ Bad: 函数太长
fun doEverything() {
    // 100+ 行代码
}
```

#### 空值处理

```kotlin
// ✅ Good: 使用安全调用
val length = text?.length ?: 0

// ✅ Good: 使用 let
file?.let { 
    compile(it)
}

// ✅ Good: 使用 require/check
fun compile(file: File) {
    require(file.exists()) { "File not found: $file" }
    // 编译逻辑
}

// ❌ Bad: 使用 !!
val length = text!!.length  // 可能抛出 NPE
```

## 开发工作流

### 功能开发流程

```
1. 创建功能分支
   git checkout -b feat/my-feature

2. 开发功能
   - 编写代码
   - 添加测试
   - 更新文档

3. 本地测试
   ./gradlew test
   ./gradlew connectedAndroidTest

4. 提交代码
   git add .
   git commit -m "feat: add my feature"

5. 推送分支
   git push origin feat/my-feature

6. 创建 Pull Request
```

### 分支策略

- `main` - 主分支，稳定版本
- `develop` - 开发分支
- `feat/*` - 功能分支
- `fix/*` - 修复分支
- `docs/*` - 文档分支

### 代码审查

提交 PR 前确保：
- [ ] 代码符合规范
- [ ] 通过所有测试
- [ ] 添加必要的文档
- [ ] 更新 CHANGELOG

## 提交规范

### Commit Message 格式

使用 Conventional Commits 规范：

```
<type>(<scope>): <subject>

<body>

<footer>
```

#### Type 类型

- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式（不影响功能）
- `refactor`: 重构
- `perf`: 性能优化
- `test`: 测试相关
- `build`: 构建系统
- `ci`: CI 配置
- `chore`: 其他杂项

#### 示例

```bash
# 新功能
git commit -m "feat(compiler): add xmake support"

# 修复 bug
git commit -m "fix(native): fix memory leak in compiler"

# 文档更新
git commit -m "docs(readme): update quick start guide"

# 重构
git commit -m "refactor(ui): simplify project list component"
```

### Pull Request 规范

#### PR 标题

```
[Type] Brief description
```

例如：
- `[Feature] Add xmake project support`
- `[Fix] Fix compilation crash`
- `[Docs] Update architecture documentation`

#### PR 描述模板

```markdown
## 变更说明
简要描述这个 PR 做了什么

## 变更类型
```

## 常见问题

### Q: 如何添加新的编译器支持？

A: 
1. 实现 `Compiler` 接口
2. 在 `CompileProjectUseCase` 中注册
3. 添加测试
4. 更新文档

### Q: 如何调试 JNI 崩溃？

A:
1. 查看 Logcat 中的 native crash 日志
2. 使用 `ndk-stack` 解析堆栈：
```bash
adb logcat | ndk-stack -sym app/build/intermediates/cxx/Debug/obj
```

### Q: 如何优化编译速度？

A:
- 使用增量编译
- 启用并行构建
- 使用编译缓存
- 减少不必要的依赖
