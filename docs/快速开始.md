## 环境准备

### 必需工具

1. **Android Studio** (最新稳定版)
   - 下载地址: https://developer.android.com/studio

2. **Docker Desktop** (用于构建工具链)
   - Windows: https://www.docker.com/products/docker-desktop
   - 确保 Docker 正在运行

3. **PowerShell 7+**
   - Windows 11 自带
   - 或从 Microsoft Store 安装

4. **Git**
   - 用于克隆项目

### 克隆项目

```bash
git clone https://github.com/wuxianggujun/TinaIDE.git
cd TinaIDE
```

## 构建工具链

TinaIDE 使用嵌入式 Clang/LLVM 工具链，需要先构建。

### 步骤 1: 构建 LLVM/Clang

```powershell
# 构建 arm64-v8a 版本（真机）
pwsh ./docker/llvm-build/build-local.ps1 -Abi arm64-v8a -ApiLevel 24

# 或构建 x86_64 版本（模拟器）
pwsh ./docker/llvm-build/build-local.ps1 -Abi x86_64 -ApiLevel 24
```

这个过程需要 30-60 分钟，取决于你的机器性能。

### 步骤 2: 同步到项目

```powershell
# 同步库文件和 sysroot
pwsh ./tools/sync-llvm-build.ps1 -Abi arm64-v8a -ApiLevel 24
```

这会将编译好的工具链同步到：
- `app/src/main/jniLibs/<abi>/` - 动态库
- `app/src/main/assets/sysroot.zip` - 系统根目录

## 构建应用

### 使用 Android Studio

1. 打开 Android Studio
2. 选择 "Open an Existing Project"
3. 选择 TinaIDE 目录
4. 等待 Gradle 同步完成
5. 点击 "Run" 按钮

### 使用命令行

```bash
# 构建 Debug 版本
./gradlew assembleDebug

# 安装到设备
./gradlew installDebug

# 或一步到位
./gradlew installDebug
```

## 创建项目

### 1. 启动应用

首次启动时，应用会自动解压 sysroot.zip 到内部存储。

### 2. 创建新项目

1. 点击主界面的 "新建项目" 按钮
2. 选择项目类型：
   - **C++ (单文件)**: 简单的单文件项目
   - **xmake 项目**: 使用 xmake 构建系统（实验性）
3. 输入项目名称
4. 选择保存位置
5. 点击 "创建"

### 3. 项目结构

#### 单文件项目
```
my_project/
└── main.cpp
```

#### xmake 项目
```
my_project/
├── xmake.lua
├── src/
│   └── main.cpp
└── include/
```

## 编译代码

### 单文件编译

1. 打开 `.cpp` 或 `.c` 文件
2. 点击工具栏的 "编译" 按钮
3. 查看输出面板的编译结果

### xmake 项目编译

1. 确保项目根目录有 `xmake.lua`
2. 点击 "编译" 按钮
3. 系统自动检测并使用 xmake 构建
4. 查看详细的构建日志

### 编译选项

在设置中可以配置：
- 优化级别 (-O0, -O1, -O2, -O3)
- C++ 标准 (C++11, C++14, C++17, C++20)
- 调试信息 (-g)
- 警告级别 (-Wall, -Wextra)

## 常见问题

### Q: 构建工具链失败

**A**: 确保 Docker 正在运行，并且有足够的磁盘空间（至少 20GB）。

### Q: 应用启动后黑屏

**A**: 检查 Logcat，可能是 sysroot 解压失败。尝试重新安装应用。

### Q: 编译失败，找不到头文件

**A**: 确保已执行同步脚本：
```powershell
pwsh ./tools/sync-llvm-build.ps1 -Abi arm64-v8a -ApiLevel 24
```

### Q: xmake 项目编译失败

**A**: 检查：
1. `xmake.lua` 语法是否正确
2. sysroot 是否完整
3. 查看输出面板的详细错误信息

### Q: 模拟器上无法执行

**A**: 确保使用 x86_64 架构的工具链：
```powershell
pwsh ./docker/llvm-build/build-local.ps1 -Abi x86_64 -ApiLevel 24
pwsh ./tools/sync-llvm-build.ps1 -Abi x86_64 -ApiLevel 24
```

### Q: 如何查看详细日志

**A**: 
1. 在应用中打开 "输出" 面板
2. 或使用 adb logcat：
```bash
adb logcat | grep -i tina
```
